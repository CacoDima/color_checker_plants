# color-checker-plants

Надёжное определение цветовой мишени через ArUco-маркеры на углах — без хрупкой
детекции контуров. Работает при поворотах до 45°, частичном перекрытии и
умеренном размытии.

## Карточки

| X-Rite Classic 24 патча | Градационный серый 50 патчей |
|:---:|:---:|
| ![Classic 24](color_checker_plants/card_preview/classic.png) | ![Gray ramp 50](color_checker_plants/card_preview/gray_50.png) |
| ArUco ID 0–3 · 6 × 4 · A4 | ArUco ID 4–7 · 10 × 5 · A4 |

Напечатать карточку:
```bash
python scripts/generate_card.py --preset classic  --output card.png --dpi 300
python scripts/generate_card.py --preset gray_50  --output gray_ramp.png --dpi 300
```

## Шо да как

Идея такая: **мы не ищем саму мишень — мы вклеиваем в её
углы четыре уникальных маркера и ищем их**. Маркеры ArUco распознаются
надёжно при поворотах до 45°, частичном перекрытии и умеренном размытии.
После нахождения маркеров делается коррекция перспективы и дальше мы извлекаем цвета из сетки

## ArUco-маркеры — подробно

### Что такое ArUco

ArUco (Augmented Reality University of Córdoba) — это бинарные квадратные
маркеры, разработанные специально для компьютерного зрения. Каждый маркер
представляет собой чёрно-белую сетку из N×N битов внутри чёрной рамки
на белом фоне. OpenCV умеет обнаруживать их с точностью до субпикселя и
однозначно определять идентификатор (ID) и ориентацию каждого.

### Какой словарь использовать

ArUco-словарь — это набор допустимых кодов маркеров, подобранных так,
чтобы любые два кода из набора как можно сильнее отличались друг от друга
(расстояние Хэмминга). Чем больше расстояние — тем устойчивее
распознавание при дефектах печати, зернистости и размытии.

| Словарь | Сетка | ID в наборе | Расст. Хэмминга | Рекомендация |
|---|---|---|---|---|
| `DICT_4X4_50` | 4×4 бита | 50 | 6 | слабовато |
| **`DICT_5X5_50`** | **5×5 бит** | **50** | **8** | **✓ используем** |
| `DICT_5X5_100` | 5×5 бит | 100 | 7 | чуть хуже |
| `DICT_6X6_250` | 6×6 бит | 250 | 10 | избыточно для печати |
| `DICT_ARUCO_ORIGINAL` | 5×5 эквив. | 1024 | ~10 | слишком крупные |

**В проекте используется `DICT_5X5_50`, ID 0, 1, 2, 3.**

Почему именно `DICT_5X5_50`:
- Расстояние Хэмминга 8 — значит, маркер остаётся распознаваемым, даже
  если до 4 битов из 25 испорчены (шум, царапина, плохая тонер-картридж).
- 50 кодов достаточно для любого применения — нам нужно всего 4.
- Физический размер маркера можно сделать компактным: при 300 DPI маркер
  20 мм даёт **236 пикселей / 7 модулей** (5 бит + по 1 рамке с каждой
  стороны) = **33 пикселя на модуль** — отлично читается.
- В отличие от `DICT_4X4_50`, при перекосе 40° и умеренном сжатии JPEG
  правильно распознаются все 4 маркера.

### Какие конкретно ID использовать

```
[ID=0]──────────────────────[ID=1]
│                                │
│         сетка цветов           │
│                                │
[ID=3]──────────────────────[ID=2]
```

| Угол карточки | ID маркера | Внутренний угол маркера (→ угол сетки) |
|---|---|---|
| Верхний левый  | 0 | правый нижний (индекс 2) → TL сетки  |
| Верхний правый | 1 | левый нижний  (индекс 3) → TR сетки  |
| Нижний правый  | 2 | левый верхний (индекс 0) → BR сетки  |
| Нижний левый   | 3 | правый верхний (индекс 1) → BL сетки |

OpenCV возвращает углы каждого маркера в порядке: [TL=0, TR=1, BR=2, BL=3].
Программа автоматически выбирает нужный угол в зависимости от ID.

### Физический размер маркеров

Рекомендуемый размер для карточки формата A4:

| Дистанция съёмки | Мин. размер маркера |
|---|---|
| 30–50 см (стол, съёмка сверху) | 15 мм |
| 50–100 см | 20 мм ← оптимально для A4 |
| 1–2 м (напольная установка) | 30–40 мм |

Правило: маркер должен занимать **не менее 20–30 пикселей** по стороне
на итоговом снимке. При съёмке 12 МП камерой (4000×3000 пикс.) с
дистанции 80 см маркер 20 мм займёт примерно 80 пикселей — отлично.

### Требования к печати

- **Бумага:** матовая. Глянцевая даёт зеркальные блики, которые путают
  детектор — он принимает пересвет за белый модуль маркера.
- **Тонер/чернила:** максимальная плотность. Серые пиксели вместо чёрных
  резко снижают контраст, расстояние Хэмминга «съедается».
- **Разрешение:** минимум 150 DPI для маркера от 20 мм. Рекомендуется 300 DPI.
- **«Тихая зона»:** вокруг каждого маркера должно быть белое поле шириной
  не менее одного модуля маркера (≈ 3–4 мм). Скрипт `generate_card.py`
  учитывает это автоматически.
- **Ламинирование:** если карточка будет использоваться на открытом воздухе —
  рекомендуется глянцевый ламинат (не путать с глянцевой бумагой):
  при диффузном освещении блики незначительны, зато защита от влаги важна.

## Быстрый старт

### Шаг 1 — Сгенерировать карточку для печати

```bash
python scripts/generate_card.py --output card.png --dpi 300 --width_mm 210
```

Флаги:

| Флаг | По умолчанию | Описание |
|---|---|---|
| `--output` | `color_checker_card.png` | Имя выходного файла |
| `--dpi` | 300 | Разрешение (DPI). Печатать именно на этом DPI |
| `--width_mm` | 210 | Ширина карточки в мм (210 = A4) |
| `--border_mm` | 10 | Белое поле по краям |
| `--marker_mm` | 20 | Размер каждого ArUco-маркера |

### Шаг 2 — Напечатать и подготовить

1. Печатать `card.png` **строго при указанном DPI** (в настройках принтера
   выбрать «фактический размер» или указать DPI явно — не «вписать в лист»).
2. Аккуратно обрезать по внешнему краю белого поля.
3. Приклеить на плотную подложку или заламинировать.

### Шаг 3 — Сфотографировать мишень

- Все 4 маркера должны быть **полностью в кадре**.
- Угол к плоскости карточки — до 45° (лучше до 30°).
- Равномерное рассеянное освещение. Избегайте прямых источников света,
  дающих блики на поверхности карточки.
- Никакого motion blur (сниматься на штативе или с высокой выдержкой).

### Шаг 4 — Обработать снимок

```bash
# Определить мишень, подобрать коррекцию, сохранить результат
python scripts/process_image.py photo.jpg

# С отладочными изображениями (разметка маркеров + выровненная сетка)
python scripts/process_image.py photo.jpg --debug

# Выбрать метод подбора матрицы
python scripts/process_image.py photo.jpg --method poly
```

## CLI — подробные опции

### Определение + коррекция одного снимка

```bash
python scripts/process_image.py <фото.jpg> [опции]
```

| Опция | По умолчанию | Описание |
|---|---|---|
| `--output FILE` | `<имя>_corrected.jpg` | Куда сохранить результат |
| `--method` | `poly` | Метод матрицы: `matrix`, `poly`, `root_poly` |
| `--extraction` | `sigma_clip` | Метод извлечения цвета из ячеек |
| `--sigma` | `2.5` | Порог отсечения для sigma_clip |
| `--save-model FILE` | — | Сохранить модель коррекции в `.npz` |
| `--debug` | — | Сохранить overlay маркеров и выровненную сетку |

### Пакетная обработка с сохранённой моделью

```bash
# Один раз подобрать модель по эталонному снимку:
python scripts/process_image.py reference.jpg --save-model shoot.npz

# Применить ко всей серии снимков:
python scripts/process_image.py apply --model shoot.npz *.jpg
```

## Методы подбора матрицы цветокоррекции

| Метод | Термины | Когда использовать |
|---|---|---|
| `matrix` | 3×3 + смещение (линейный) | Хорошее освещение, студия |
| `poly` (по умолч.) | 11 термов (Cheung 2004) | Универсально, репортажная съёмка |
| `root_poly` | 13 термов (Finlayson 2015) | Широкий охват, HDR-камеры |

**Cheung 2004** расширяет цветовой вектор [R, G, B] до
[R, G, B, RG, RB, GB, R², G², B², RGB, 1] — перекрёстные члены позволяют
исправлять мягкие нелинейности канала, которые неизбежны при съёмке в
переменном освещении. 24 патча дают 24 уравнения для 11 неизвестных —
система переопределена, решается методом наименьших квадратов.

**Root-polynomial (Finlayson 2015)** вместо произведений использует их
квадратные корни: `√(RG)` вместо `RG`. Это улучшает числовую
обусловленность при экстремальных значениях каналов и снижает
переобучение.

---

## Методы извлечения цвета

| Метод | Описание |
|---|---|
| `sigma_clip` | **По умолчанию.** Итеративно отбрасывает пиксели дальше 2.5σ от среднего, усредняет оставшиеся. Устойчив к пыли, бликам, граничным засветкам между ячейками. |
| `median` | Медиана по каждому каналу. Чуть медленнее, хорошо работает при грубых загрязнениях. |
| `trimmed_mean` | Обрезанное среднее (отбрасывает 10% с каждого края гистограммы). |

## Градационный чекер (50 серых патчей)

Помимо стандартного 24-патч колорчекера в библиотеке есть большой градационный
чекер: **50 нейтральных патчей** от чистого чёрного до чистого белого в сетке 10 × 5.

Маркеры **ID 4, 5, 6, 7** (DICT\_5X5\_50) — специально отличаются от стандартного
чекера (ID 0–3), чтобы оба можно было держать в одном кадре одновременно. TODO я хз будет ли это работать сразу с двумя

### Шаг 1 — Напечатать карточку

```bash
python scripts/generate_card.py --preset gray_50 --output gray_ramp.png --dpi 300
```

### Шаг 2 — Использовать в коде

```python
import cv2
from color_checker_plants import detect_color_checker, fit_correction, apply_correction
from color_checker_plants.templates.grayscale_ramp_50 import (
    REFERENCE_SRGB, GRID_COLS, GRID_ROWS, MARKER_IDS
)

image = cv2.cvtColor(cv2.imread("photo.jpg"), cv2.COLOR_BGR2RGB).astype("float32") / 255

result = detect_color_checker(
    image,
    grid_cols=GRID_COLS,          # 10
    grid_rows=GRID_ROWS,          # 5
    reference_colors=REFERENCE_SRGB,
    marker_ids=MARKER_IDS,        # IDs 4-7
)

if result is None:
    print("Маркеры не найдены — все 4 угла должны быть в кадре")
else:
    model = fit_correction(result.measured_colors, result.reference_colors)
    corrected = apply_correction(image, model)
```

### Параметры карточки

| Параметр | Значение |
|---|---|
| Патчей | 50 (сетка 10 × 5) |
| Диапазон | 0–255, равный шаг |
| Маркеры | DICT\_5X5\_50, ID 4, 5, 6, 7 |
| Рекомендуемый размер | A4 ширина, 300 DPI |

## Расширение на другие мишени

Чтобы использовать другую цветовую мишень (например, X-Rite SG 140 патчей
или собственную):

1. Создать `color_checker_plants/templates/my_checker.py` по образцу
   `colorchecker_24.py`: задать `REFERENCE_SRGB`, `GRID_COLS`, `GRID_ROWS`,
   `MARKER_IDS`, `MARKER_INNER_CORNER_INDEX`.
2. Передать параметры явно:
   ```python
   detect_color_checker(
       image,
       grid_cols=14,
       grid_rows=10,
       reference_colors=MY_REFERENCE,
       marker_ids=MY_MARKER_IDS,
   )
   ```

## Частые проблемы

| Проблема | Причина | Решение |
|---|---|---|
| «Could not find all 4 ArUco markers» | Маркер вне кадра или перекрыт | Убедиться, что все 4 угла видны |
| Детектор находит маркеры, но цвета кривые | Перепутан порядок углов | Проверить, что ID 0 — левый верхний, 2 — правый нижний |
| Средние ΔE > 0.05 после коррекции | Блик на карточке при съёмке | Снимать при рассеянном свете, без прямого источника |
| Маркеры не распознаются на телефонных снимках | JPEG-артефакты | Снимать в RAW или минимальное сжатие JPEG |
| `ImportError: cv2.aruco не найден` | Установлен обычный `opencv-python` | `pip uninstall opencv-python && pip install opencv-contrib-python` |
